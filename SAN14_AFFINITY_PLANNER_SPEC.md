# 삼국지14 장수 편성기 기획/구현 정리

## 1. 프로젝트 개요

삼국지14의 장수 친애 관계, 능력치, 개성 데이터를 기반으로 장수 조합과 분파 편성을 추천하는 정적 웹 앱이다.

주요 목적은 두 가지다.

1. 특정 장수를 중심으로 가능한 친애 네트워크와 분파 후보를 확인한다.
2. 보유 장수 목록을 입력하면, 분파당 10명 내외로 효율적인 조합을 자동 편성한다.

앱은 서버 없이 동작하는 정적 웹 앱으로 구성한다. GitHub에 올린 뒤 GitHub Pages로 접속 가능하게 하고, GitHub Release는 버전 배포와 압축 파일 제공용으로 사용한다.

## 4. UX 기획

앱은 두 가지 모드를 제공한다.

```text
보유 장수 편성
중심 분석
```

### 4.1 보유 장수 편성 모드

사용자가 보유한 장수 이름을 여러 명 입력한다.

입력 예시:

```text
유비, 관우, 장비, 황충, 조운, 마초, 마대, 제갈량
```

출력:

```text
1분파
- 구성원
- 버프 중첩 B값
- 연결 근거
- 의형제 후보
- 결혼 후보(미혼 가정)

2분파
...
```

분파 인원은 기본 10명 이내로 제한한다.

### 4.2 중심 분석 모드

특정 장수 한 명을 입력한다.

입력 예시:

```text
관우
```

출력:

```text
단독 연결
짧은 경로부터 긴 경로 순으로 연결 근거 출력
```

출력 형식 예:

```text
관우 <-> 장비 ☆
관우 <-> 조운 ◎
관우 <-> 관평 ☆
관우 <-> 장비 <-> 조운 ☆◎
```

## 5. 핵심 점수 로직

### 5.1 친애 관계 점수

관계 강도는 다음처럼 계산한다.

```text
☆ = 3
◎ = 2
○ = 1
X = 제외
```

관계는 단방향/역방향/쌍방향 모두 탐색한다.

표시 방식:

```text
A -> B  : A가 B를 친애
A <- B  : B가 A를 친애
A <-> B : 양쪽 모두 친애
```

### 5.2 개성 점수

개성 점수는 셀 배경색 기준으로 계산한다.

```text
금색 개성: +5
파란색 개성: +2
빨간색 개성: +0
```

추가 보정:

```text
악주: +1
응원: +1
```

예시:

```text
응원은 파란색 개성이므로 기본 +2, 범위 버프 보정 +1, 총 +3
```

### 5.3 능력치 우선도

현재 장수 기본 전투/편성 가치는 다음 우선순위로 계산한다.

```text
1순위 primary   = max(통솔 + 무력, 통솔 + 지력) + 개성점수
2순위 secondary = 통솔 + 정치 + 개성점수
3순위 third     = 지력 + 정치 + 개성점수
4순위 fourth    = 통솔 + 개성점수
```

현재 UI에서는 주로 `primaryScore`를 표시한다.

## 6. 버프 중첩 로직

같은 분파 안에 있는 장수끼리 `☆`, `◎`, `○` 관계가 있으면 양쪽 모두 버프 1중첩을 얻는 것으로 계산한다.

예시:

```text
관우 <-> 장료
관우 <-> 서황
```

버프 중첩:

```text
관우 B2
장료 B1
서황 B1
```

의형제나 결혼은 기본 편성 점수에 직접 반영하지 않고, 추천 후보로 별도 표시한다.

## 7. 의형제 추천 로직

의형제는 최대 3명까지 가능하다고 가정한다.

추천 기준:

```text
1. 같은 분파 안에서 3인 조합을 만든다.
2. 세 명 사이의 기존 친애 연결 수를 계산한다.
3. 이미 3/3으로 완성된 삼각형은 추천하지 않는다.
4. 기존 연결이 2/3인 조합을 우선 추천한다.
5. 능력치 합이 높은 조합을 우선한다.
```

출력 예:

```text
관우 + 조운 + 관흥 / 기존연결 2/3, 예상스택 +2
황충 + 엄안 + 법정 / 기존연결 2/3, 예상스택 +2
장비 + 황충 + 법정 / 기존연결 1/3, 예상스택 +4
```

의형제 추천의 목적은 이미 어느 정도 연결된 조합을 완전한 삼각형으로 닫아 버프 중첩을 보강하는 것이다.

## 8. 결혼 추천 로직

기본 편성에서는 여무장을 모두 기혼 상태로 취급한다.

따라서 결혼은 기본 분파 점수와 버프 중첩 계산에 반영하지 않는다.

다만 사용자가 미혼 상태를 가정할 수 있도록 별도 추천 섹션을 제공한다.

```text
결혼 후보(미혼 가정)
```

추천 기준:

```text
1. 같은 분파 안의 남무장 + 여무장 조합만 후보로 본다.
2. 기존 친애가 없는 경우 신규 연결 가치가 높다.
3. 여무장이 응원/악주를 가진 경우 추가 가치를 준다.
4. 기본 편성 점수에는 반영하지 않는다.
```

출력 예:

```text
노식 + 후씨 / 신규연결, 예상가치 +4
제갈량 + 미씨 / 신규연결, 예상가치 +4
황충 + 마운록 / 신규연결, 예상가치 +2
```

## 9. 보유 장수 편성 알고리즘

완전탐색은 보유 장수가 많아지면 경우의 수가 급격히 커진다. 따라서 현재 구현은 정적 웹 앱에서 빠르게 동작하도록 휴리스틱 방식으로 고정한다.

### 9.1 기본 흐름

```text
1. 입력된 장수명을 ID로 해석한다.
2. 동명이인은 현재 명단과의 연결도가 높은 후보를 자동 선택한다.
3. 별(☆) 관계로 강하게 연결된 컴포넌트를 먼저 만든다.
4. 각 컴포넌트를 최대 분파 인원 이하로 유지한다.
5. 남은 장수를 기존 분파에 넣었을 때의 marginal gain을 계산한다.
6. 친애 연결, 관계 강도, 장수 점수, 개성 점수를 기준으로 배정한다.
7. 어떤 분파에도 충분히 어울리지 않으면 새 분파를 만든다.
8. 각 분파를 점수순으로 정렬해 출력한다.
```

### 9.2 분파 점수

분파 점수는 다음 요소를 합산한다.

```text
장수 primaryScore 합
버프 중첩 수
친애 관계 강도
```

현재 구현상 대략적인 계산:

```text
분파점수 = primaryScore 합 + 버프중첩합 * 15 + 관계강도합 * 10
```

이 점수는 절대적인 게임 공식이 아니라 추천 정렬용 내부 지표다.

### 9.3 장수 추가 평가

기존 분파에 새 장수를 넣을 때는 다음을 본다.

```text
분파 내부 친애 연결 수
관계 강도
별 관계 여부
장수 primaryScore
개성 점수
```

즉, 단순히 능력치가 높은 장수보다 기존 분파와 실제로 연결되는 장수를 우선한다.

## 10. 중심 분석 알고리즘

중심 장수 분석은 단일 장수를 기준으로 최대 4-hop까지 탐색한다.

기본 규칙:

```text
1. 중심 장수 A를 선택한다.
2. A와 직접 연결된 장수를 찾는다.
3. B -> C -> D -> E 형태로 최대 4-hop까지 탐색한다.
4. 정방향뿐 아니라 역방향 연결도 포함한다.
5. 짧은 경로부터 긴 경로 순으로 출력한다.
6. 같은 경로 안에서 이미 방문한 장수는 재방문하지 않는다.
```

출력은 hop별 섹션으로 나누지 않고, 짧은 경로부터 긴 경로 순으로 정렬한다.

## 14. 참고사항과 한계

### 14.1 동명이인

삼국지14 데이터에는 같은 이름의 장수가 존재한다.

현재 구현은 입력 장수가 동명이인일 경우 현재 명단 안의 다른 장수들과 더 많이 연결되는 후보를 자동 선택한다.

필요하면 추후 UI에서 직접 선택하도록 개선할 수 있다.

```text
순욱#393
순욱#397
```

### 14.2 완전 최적화가 아님

현재 보유 장수 편성은 완전탐색 최적해가 아니라 휴리스틱 추천이다.

이유:

```text
보유 장수 수가 많아질수록 분파 배치 경우의 수가 폭증한다.
정적 웹 앱에서 즉시 반응해야 한다.
사용자는 정확한 수학적 최적해보다 해석 가능한 편성안을 원한다.
```

### 14.3 점수는 정렬용 내부 지표

분파 점수와 추천 점수는 인게임 공식이 아니다.

목적:

```text
여러 후보를 비교하기 쉽게 정렬
좋은 분파를 상단에 표시
연결 밀도와 능력치를 함께 반영
```

### 14.4 결혼 추천은 가정값

여무장은 기본적으로 기혼 처리한다.

결혼 후보는 사용자가 미혼 상태를 가정할 때 참고하는 별도 시뮬레이션이다.

따라서 결혼 후보가 출력되더라도 실제 기본 편성 점수에는 반영하지 않는다.

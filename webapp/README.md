# 삼국지14 친애 편성기

정적 웹 앱입니다. `index.html`을 브라우저에서 열면 바로 실행됩니다.

## 고정된 로직

- 친애 관계는 `☆`, `◎`, `○`만 사용하고 `X`는 제외합니다.
- 버프 중첩은 같은 집단 안의 양방향/단방향 친애 연결 수로 계산합니다.
- 장수 점수는 `max(통솔+무력, 통솔+지력) + 개성점수`를 우선값으로 사용합니다.
- 개성 점수는 금색 `+5`, 파란색 `+2`, 빨간색 `+0`입니다.
- `악주`, `응원`은 추가 `+1`을 더합니다.
- 보유 장수 편성은 `목표 집단 수`와 `집단당 최대 인원`을 함께 기준으로 삼는 휴리스틱 분배입니다.
- `집단당 최대 인원`은 기본값 10명, 입력 상한 20명입니다.
- 보유 장수 목록에 동명이인이 일부만 입력되면 `동명이인 선택` 프레임에서 후보를 직접 고릅니다.
- 동명이인 이름이 후보 수만큼 입력되면 모든 후보가 존재한다고 보고 `장소(1)`, `장소(2)`처럼 구분합니다.
- 두 조건을 동시에 만족할 수 없으면 최대 인원 기준으로 가능한 집단 수 범위로 보정합니다.
- 3인 이하 소집단이 남으면 포화 집단 교환, 빈자리 흡수, 소집단 병합 순서로 재배치를 시도합니다.
- 1인 집단은 연결 앵커 이동과 빈자리 보강까지 추가로 검토합니다.
- 재배치는 전체 점수가 오르는 경우에만 적용합니다.
- 재배치 과정은 별도 안내 없이 최종 편성 결과에만 반영합니다.
- 결과창은 `편성 결과`와 `의형제 추천` 탭으로 나뉩니다.
- `편성 결과` 탭은 집단 구성과 연결 근거만 표시합니다.
- 무장 배지는 `관우 +2`처럼 이름과 버프점수만 표시합니다.
- 연결 근거는 작은 카드로 표시하고, 첫 번째 기준 장수 순서대로 정렬합니다.
- `의형제 추천`은 3인 의형제를 임시 `☆` 연결로 가정하고, 총점과 소집단 정리가 함께 좋아지는 추천세트를 표시합니다.
- 의형제 추천의 예상 변화는 실제 변동이 있는 항목만 배지로 표시하고, 예상 편성은 카드형 목록으로 표시합니다.
- 중심 분석은 결과 탭을 표시하지 않고, 중심 장수가 친애하는 대상과 중심 장수를 친애하는 대상을 합친 전체 직접 친애 목록을 표시합니다.
- 중심 분석의 3인 이상 경로는 점수가 높은 순서로 최대 50개를 `집단 추천` 스크롤 프레임에 표시하며 탐색 단위는 `단계`로 표기합니다.
- 결혼 후보와 집단별 단순 의형제 후보는 표시하지 않습니다.

## 데이터 갱신

원본 `affinity_index.json`과 엑셀 파일이 바뀌면 아래 스크립트로 `data.js`를 다시 생성합니다.
기본값은 저장소 루트의 `affinity_index.json`과 `삼국지14_전체무장_공유(통합본)의 사본.xlsx`입니다.

```powershell
python .\webapp\build_data.py
```

다른 위치의 파일을 사용하려면 환경변수로 지정할 수 있습니다.

```powershell
$env:SAN14_AFFINITY_JSON = 'C:\path\to\affinity_index.json'
$env:SAN14_WORKBOOK = 'C:\path\to\삼국지14_전체무장_공유(통합본)의 사본.xlsx'
python .\webapp\build_data.py
```
